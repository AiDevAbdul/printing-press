## Fix #1: Update StatsCard Component (5 min) 🔴 MEDIUM PRIORITY

### Issue
`src/components/admin/StatsCard.tsx` uses non-existent CSS classes that don't map to theme variables.

### Current Code
```tsx
export default function StatsCard({ label, value, icon }: StatsCardProps) {
  return (
    <div className="bg-primary-section border border-primary-action rounded-lg p-6">
      <div className="flex items-center justify-between">
        <div>
          <p className="text-text-muted text-sm">{label}</p>
          <p className="text-3xl font-bold text-primary-action mt-2">{value}</p>
        </div>
        {icon && <div className="text-4xl">{icon}</div>}
      </div>
    </div>
  );
}
```

### Problem
- `bg-primary-section` → doesn't exist
- `border-primary-action` → doesn't exist
- `text-text-muted` → doesn't exist
- `text-primary-action` → doesn't exist

### Fixed Code
```tsx
export default function StatsCard({ label, value, icon }: StatsCardProps) {
  return (
    <div className="bg-light-bg-secondary dark:bg-dark-bg-secondary border border-light-border dark:border-dark-border rounded-lg p-6">
      <div className="flex items-center justify-between">
        <div>
          <p className="text-light-text-secondary dark:text-dark-text-secondary text-sm">{label}</p>
          <p className="text-3xl font-bold text-light-accent dark:text-dark-accent mt-2">{value}</p>
        </div>
        {icon && <div className="text-4xl">{icon}</div>}
      </div>
    </div>
  );
}
```

| `border-primary-action` | `border-light-border dark:border-dark-border` | `--color-light-border`, `--color-dark-border` |

| `text-primary-action` | `text-light-accent dark:text-dark-accent` | `--color-light-accent`, `--color-dark-accent` |

## Fix #2: Remove Redundant :root Variables (2 min) 🟡 LOW PRIORITY

### Issue
Lines 39-47 in `globals.css` define CSS variables that are already generated by Tailwind v4's `@theme` block.

### Current Code (globals.css, lines 39-47)
```css
/* Explicit CSS custom properties for light mode */
:root {
  --font-space-grotesk: 'Space Grotesk', sans-serif;
  --font-inter: 'Inter', sans-serif;
  --font-jetbrains-mono: 'JetBrains Mono', monospace;
  --color-light-accent: #86C620;
  --color-light-accent-hover: #7AB81A;
  --color-dark-accent: #A16CCB;
  --color-dark-accent-hover: #B88FD9;
}
```

### Why Remove?
- Tailwind v4 automatically generates `:root` variables from `@theme` block
- These are duplicates that add unnecessary CSS
- Maintenance burden: if you update `@theme`, you'd need to update `:root` too
- Saves ~50 bytes of CSS

### Fixed Code
Delete lines 39-47 entirely. The `@theme` block (lines 5-36) already provides all these variables.

### Steps
1. Open `src/app/globals.css`
2. Delete lines 39-47 (the entire `:root { }` block)
3. Verify the file still compiles: `npm run build`
4. Check that colors still work in browser

### Verification
After deletion:
- ✅ Build completes successfully
- ✅ All colors render correctly
- ✅ Dark mode still works
- ✅ No console errors
